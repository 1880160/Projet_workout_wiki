# Modèles de conception qui seront utilisés

## Composite

Le modèle composite sera utilisé pour structurer nos routes. Par conséquent, les sous-routes seront contenues dans des sous-dossiers dans une structure similaire aux routes de l'api/frontend.

## Singleton

NestJs utilise le singleton par défaut dans son dépôt.

## Décorateur

Le décorateur sera utilisé pour injecter du code dans les fonctions et les routes
afin de réduire considérablement la quantité de code et d'améliorer la lisibilité du code.
