# Features
Here's the list of features which are planned or debated.

## Planned features
These are the features which will be focused on since they're confirmed to be necessary or wanted for the application.
Beware that bugfixes and code refactoring of the application are not included in here.


### The "Play/Execute" workout page
This page is intented to be used while doing a workout. It will have a simple UI which display the exercises for a workout session. It will have features suchs as a timer, a way to mark exercises as finished and automatic exercise switching for circuit type workout.


### Mobile app with capacitorjs
The frontend will receive a mobile version using capacitor. This will ease the use of the app when using it away from a computer. Moreover, capacitor doesn't require rewritting the web implementation, hence it will be easy to sync with the web ui.




## Debated features
These are features which are not confirmed yet. But are seriously considered.


### Making the app client first
This will allow to make the app not require a backend to work. Only requiring the backend for syncing.
Rxdb, Powersync or Electric Sql could be used for syncing a local sqlite database with a bigger database.
